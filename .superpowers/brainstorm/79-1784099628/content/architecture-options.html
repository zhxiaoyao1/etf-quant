系统架构方案对比

PWA 移动端 + 免费数据 + 因子权重自学习

A

纯前端 · 浏览器全搞定

优势

  • 零服务器成本，静态页面即可
  • 部署最简单，扔到 GitHub Pages 就能用
  • 开箱即用，无需任何后端知识

劣势

  • 浏览器端做量化计算性能有限
  • 数据获取受浏览器跨域限制
  • 数据无法持久化（清除缓存就丢失）
  • 策略回测、参数优化等重计算几乎不可行
B

Python 后端 + PWA 前端 ⭐ 推荐

优势

  • Python 是量化领域标准语言，生态最全
  • 后端处理数据抓取、指标计算、回测、优化
  • PWA 前端只负责展示和交互，流畅轻量
  • 数据持久化在本地 SQLite，安全可控
  • 可跑在个人电脑/NAS/云服务器上

劣势

  • 需要后端服务保持运行（电脑/NAS/低配云服务器）
  • 部署比纯前端多一步
C

Serverless 全云方案

优势

  • 无需维护服务器，按量付费
  • 天然高可用，随时随地访问
  • 数据在云端，不依赖本地设备

劣势

  • 有月度费用，用量上去后成本不可控
  • 冷启动延迟影响体验
  • 数据安全依赖第三方
  • 对入门用户来说架构复杂度偏高
